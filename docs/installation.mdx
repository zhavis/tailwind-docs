# Installation

Tailwind CSS can be installed in multiple ways depending on your project type: via **npm** (recommended), **CDN** (quick prototyping), or using **CLI tools**.  

---

## 1. Using npm (Recommended for Projects)

This is the recommended method for modern projects because it allows full customization and integration with build tools.

### Step 1: Initialize your project
```bash
# Create a new project folder and initialize npm
mkdir my-project
cd my-project
npm init -y
````

### Step 2: Install Tailwind CSS and its dependencies

```bash
npm install -D tailwindcss postcss autoprefixer
```

### Step 3: Generate configuration files

```bash
npx tailwindcss init -p
```

This will create two files:

* `tailwind.config.js` → Tailwind configuration
* `postcss.config.js` → PostCSS configuration

---

### Step 4: Configure Tailwind to remove unused CSS

Edit `tailwind.config.js` to specify your template paths:

```js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./src/**/*.{html,js,ts,jsx,tsx}", // adjust to your project structure
  ],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

---

### Step 5: Include Tailwind in your CSS

Create a CSS file (e.g., `src/styles.css`) and add:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

---

### Step 6: Build your CSS

Add a build script in `package.json`:

```json
"scripts": {
  "build:css": "tailwindcss -i ./src/styles.css -o ./dist/styles.css --watch"
}
```

Then run:

```bash
npm run build:css
```

Include the generated CSS in your HTML:

```html
<link href="./dist/styles.css" rel="stylesheet">
```

## 2. Using CDN (Quick Prototyping)

For small projects or quick testing, you can use the Tailwind CDN without installing npm packages:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Tailwind CDN Example</title>
  <link href="https://cdn.jsdelivr.net/npm/tailwindcss@3.3.3/dist/tailwind.min.css" rel="stylesheet">
</head>
<body>
  <div class="bg-blue-500 text-white p-4">
    Hello, Tailwind!
  </div>
</body>
</html>
```

**Note:** Using CDN is convenient but **not recommended for production**, as it prevents purging unused CSS and reduces performance optimization.

---

## 3. Using Tailwind CLI

You can also use Tailwind’s standalone CLI tool for quick builds without a full Node.js setup:

```bash
# Install globally (optional)
npm install -g tailwindcss

# Initialize Tailwind config
tailwindcss init

# Build CSS
tailwindcss -i ./src/input.css -o ./dist/output.css --watch
```

---

## 4. Integrating with Frameworks

Tailwind integrates easily with popular frameworks:

* **Next.js**

  * Create a new Next.js project
  * Install Tailwind
  * Configure `tailwind.config.js` and import CSS in `pages/_app.js`
* **React**

  * Install Tailwind via npm
  * Import Tailwind CSS in `index.js` or `App.js`
* **Vue, Angular, Laravel, etc.**

  * Tailwind provides official guides for framework integration: [Tailwind Docs](https://tailwindcss.com/docs/installation)

---

**Summary:**

* **npm setup:** Recommended for real projects and full customization.
* **CDN:** Quick testing or small prototypes.
* **CLI:** Lightweight build tool without full Node.js project.
* **Framework integration:** Tailwind works seamlessly with modern web frameworks.
