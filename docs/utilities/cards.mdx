# Cards

Cards are versatile containers for displaying content and actions about a single subject. Tailwind CSS makes it easy to create beautiful, responsive cards with consistent styling.

---

## 1. Basic Card Structure

### Simple Card
```html
<div class="max-w-sm rounded overflow-hidden shadow-lg bg-white">
  <div class="px-6 py-4">
    <div class="font-bold text-xl mb-2">Card Title</div>
    <p class="text-gray-700 text-base">
      This is a simple card example with basic styling.
    </p>
  </div>
</div>
```

### Card with Image

## 10. Card Component (React Example)

```jsx
// Card.jsx
export default function Card({
  children,
  title,
  subtitle,
  image,
  imageAlt,
  actions,
  footer,
  hoverable = false,
  className = '',
  ...props
}) {
  const baseClasses = "bg-white rounded-lg shadow";
  const hoverClasses = hoverable ? "hover:shadow-xl transition-shadow duration-300" : "";
  
  return (
    <div className={`${baseClasses} ${hoverClasses} ${className}`} {...props}>
      {/* Image */}
      {image && (
        <img 
          className="w-full h-48 object-cover rounded-t-lg" 
          src={image} 
          alt={imageAlt || ''} 
        />
      )}
      
      {/* Header */}
      {(title || subtitle) && (
        <div className="px-6 pt-6">
          {title && <h3 className="text-xl font-semibold text-gray-800">{title}</h3>}
          {subtitle && <p className="text-gray-600 mt-1">{subtitle}</p>}
        </div>
      )}
      
      {/* Content */}
      <div className="p-6">
        {children}
      </div>
      
      {/* Actions */}
      {actions && (
        <div className="px-6 pb-6">
          {actions}
        </div>
      )}
      
      {/* Footer */}
      {footer && (
        <div className="px-6 py-4 border-t border-gray-200 bg-gray-50 rounded-b-lg">
          {footer}
        </div>
      )}
    </div>
  );
}
```

**Usage:**
```jsx
<Card 
  title="Product Card"
  subtitle="Premium Plan"
  image="https://via.placeholder.com/400x200"
  hoverable
  actions={
    <button className="bg-blue-500 text-white px-4 py-2 rounded">
      Buy Now
    </button>
  }
>
  <p>Card content goes here.</p>
</Card>
```

## 11. Best Practices

### Do:
- Maintain consistent padding and spacing
- Use semantic HTML structure
- Ensure proper contrast ratios
- Add hover/focus states for interactive cards
- Make cards accessible (proper headings, ARIA labels)
- Optimize images for card thumbnails

### Don't:
- Overcrowd cards with too much information
- Use cards for primary navigation
- Forget about mobile touch targets
- Use low-contrast text on colored backgrounds
- Mix too many different card styles in one view

## 12. Accessibility Considerations

### Semantic Structure
```html
<article class="bg-white rounded-lg shadow" aria-labelledby="card-title-1">
  <header class="p-6">
    <h3 id="card-title-1" class="text-lg font-semibold">Card Title</h3>
  </header>
  <div class="px-6 pb-6">
    <p>Card content with proper semantic structure.</p>
  </div>
</article>
```

### Keyboard Navigation
```html
<div role="button" tabindex="0" 
     class="bg-white rounded-lg shadow focus:outline-none focus:ring-2 focus:ring-blue-500 cursor-pointer">
  <div class="p-6">
    <h3>Clickable Card</h3>
    <p>Accessible with keyboard navigation.</p>
  </div>
</div>
```

## Summary

Cards are essential UI components for organizing and presenting content. With Tailwind:

1. **Start simple** with basic card structures
2. **Enhance with utility classes** for styling
3. **Make them interactive** with hover and focus states
4. **Ensure responsiveness** for all screen sizes
5. **Prioritize accessibility** with proper semantic markup

Remember to keep cards focused on a single topic, use consistent styling, and maintain good information hierarchy within each card.
