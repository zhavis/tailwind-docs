# Components

While Tailwind CSS is known for its utility-first approach, it also provides ways to create reusable components. There are several methods to build components with Tailwind, from simple utility composition to full component extraction.

---

## 1. Utility-Based Components

The simplest way to create components is by **composing utility classes** directly in your HTML:

```html
<!-- Button Component -->
<button class="bg-blue-500 hover:bg-blue-600 text-white font-bold py-2 px-4 rounded">
  Primary Button
</button>

<!-- Card Component -->
<div class="max-w-sm rounded overflow-hidden shadow-lg bg-white">
  <img class="w-full" src="image.jpg" alt="Card image">
  <div class="px-6 py-4">
    <h3 class="font-bold text-xl mb-2">Card Title</h3>
    <p class="text-gray-700 text-base">
      Card description goes here.
    </p>
  </div>
</div>

<!-- Alert Component -->
<div class="bg-yellow-100 border-l-4 border-yellow-500 text-yellow-700 p-4">
  <p>This is a warning alert!</p>
</div>
```

**Advantages:**
- No extra CSS needed
- Easy to customize on the fly
- Perfect for one-off or slightly different components

---

## 2. Component Extraction with `@apply`

For frequently reused components, extract utilities into custom CSS classes using `@apply`:

```css
/* styles.css */
.btn {
  @apply font-bold py-2 px-4 rounded;
}

.btn-primary {
  @apply bg-blue-500 text-white;
}

.btn-primary:hover {
  @apply bg-blue-600;
}

.card {
  @apply max-w-sm rounded overflow-hidden shadow-lg bg-white;
}

.card-image {
  @apply w-full h-48 object-cover;
}

.card-content {
  @apply px-6 py-4;
}

.card-title {
  @apply font-bold text-xl mb-2;
}
```

**Usage in HTML:**
```html
<button class="btn btn-primary">Primary Button</button>

<div class="card">
  <img class="card-image" src="image.jpg" alt="Card">
  <div class="card-content">
    <h3 class="card-title">Card Title</h3>
    <p>Card content goes here.</p>
  </div>
</div>
```

**Best practices for `@apply`:**
- Use for truly reusable components
- Keep it minimal - don't extract everything
- Don't use `@apply` for one-off utilities

## 3. Component Libraries with Tailwind

Popular component libraries built with Tailwind:

### **Official:**
- [Tailwind UI](https://tailwindui.com/) - Paid official component library
- [Headless UI](https://headlessui.dev/) - Unstyled accessible components

### **Open Source:**
- [DaisyUI](https://daisyui.com/) - Component library for Tailwind
- [Flowbite](https://flowbite.com/) - Open-source component library
- [Meraki UI](https://merakiui.com/) - Beautiful Tailwind components

## 5. Component Patterns

### 1. **Variant Pattern**
```html
<!-- Using different variants -->
<button class="btn btn-primary">Primary</button>
<button class="btn btn-secondary">Secondary</button>
<button class="btn btn-danger">Danger</button>

<style>
.btn {
  @apply font-bold py-2 px-4 rounded;
}
.btn-primary {
  @apply bg-blue-500 text-white hover:bg-blue-600;
}
.btn-secondary {
  @apply bg-gray-500 text-white hover:bg-gray-600;
}
.btn-danger {
  @apply bg-red-500 text-white hover:bg-red-600;
}
</style>
```

### 2. **Size Pattern**
```html
<button class="btn btn-sm">Small</button>
<button class="btn btn-md">Medium</button>
<button class="btn btn-lg">Large</button>
```

### 3. **State Pattern**
```html
<button class="btn" disabled>Disabled</button>
<button class="btn loading">Loading...</button>
<button class="btn btn-active">Active</button>
```

## 6. Component Architecture

### Atomic Design with Tailwind
1. **Atoms**: Basic HTML elements with utilities
   ```html
   <button class="btn">Button</button>
   <input class="input" type="text">
   ```

2. **Molecules**: Groups of atoms
   ```html
   <div class="search-form">
     <input class="input" type="search" placeholder="Search...">
     <button class="btn">Search</button>
   </div>
   ```

3. **Organisms**: Complex components
   ```html
   <header class="header">
     <nav class="nav">...</nav>
     <div class="search-bar">...</div>
     <div class="user-menu">...</div>
   </header>
   ```

### Example React Component:
```jsx
// Button.jsx
export default function Button({ 
  children, 
  variant = 'primary', 
  size = 'md',
  ...props 
}) {
  const baseClasses = "font-medium rounded transition-colors";
  
  const variants = {
    primary: "bg-blue-600 text-white hover:bg-blue-700",
    secondary: "bg-gray-200 text-gray-800 hover:bg-gray-300",
    danger: "bg-red-600 text-white hover:bg-red-700",
  };
  
  const sizes = {
    sm: "px-3 py-1 text-sm",
    md: "px-4 py-2",
    lg: "px-6 py-3 text-lg",
  };
  
  const classes = `${baseClasses} ${variants[variant]} ${sizes[size]}`;
  
  return (
    <button className={classes} {...props}>
      {children}
    </button>
  );
}
```

**Usage:**
```jsx
<Button variant="primary" size="lg">Large Primary</Button>
<Button variant="secondary" size="sm">Small Secondary</Button>
```

## 8. Best Practices

### Do:
- Start with utility classes, extract to components when reused
- Keep components small and focused
- Use meaningful names for extracted classes
- Maintain consistent spacing and typography
- Document component usage and variants

### Don't:
- Over-extract with `@apply`
- Mix component classes with utility classes unnecessarily
- Create too many variants
- Forget about accessibility

## Summary

Tailwind components can be built in multiple ways:

1. **Utility-based**: Direct composition in HTML (simplest)
2. **`@apply` extraction**: CSS-based reusable classes
3. **Component libraries**: Pre-built solutions like DaisyUI
4. **Framework components**: React/Vue components with dynamic classes

The key is to **start simple** with utilities and **extract patterns** as they become repeated. This maintains the flexibility of Tailwind while providing the reusability of traditional component systems.
