# Buttons

Buttons are fundamental UI components that users interact with. Tailwind CSS makes it easy to create beautiful, responsive buttons with consistent styling across your application.

## 10. Button Component (React Example)

```jsx
// Button.jsx
export default function Button({
  children,
  variant = 'primary',
  size = 'md',
  disabled = false,
  loading = false,
  icon = null,
  iconPosition = 'left',
  className = '',
  ...props
}) {
  const baseClasses = 'font-medium rounded transition-colors focus:outline-none focus:ring-2';
  
  const variants = {
    primary: 'bg-blue-500 text-white hover:bg-blue-600 focus:ring-blue-300',
    secondary: 'bg-gray-200 text-gray-800 hover:bg-gray-300 focus:ring-gray-300',
    outline: 'border border-gray-300 text-gray-700 hover:bg-gray-50 focus:ring-gray-300',
    danger: 'bg-red-500 text-white hover:bg-red-600 focus:ring-red-300',
    success: 'bg-green-500 text-white hover:bg-green-600 focus:ring-green-300',
  };
  
  const sizes = {
    sm: 'text-sm py-1 px-3',
    md: 'py-2 px-4',
    lg: 'text-lg py-3 px-6',
  };
  
  const disabledClasses = disabled ? 'opacity-50 cursor-not-allowed' : '';
  
  const classes = `${baseClasses} ${variants[variant]} ${sizes[size]} ${disabledClasses} ${className}`;
  
  return (
    <button className={classes} disabled={disabled || loading} {...props}>
      {loading ? (
        <div className="flex items-center justify-center">
          <svg className="animate-spin h-4 w-4 mr-2" viewBox="0 0 24 24">
            {/* Spinner SVG */}
          </svg>
          Loading...
        </div>
      ) : (
        <div className="flex items-center justify-center">
          {icon && iconPosition === 'left' && <span className="mr-2">{icon}</span>}
          {children}
          {icon && iconPosition === 'right' && <span className="ml-2">{icon}</span>}
        </div>
      )}
    </button>
  );
}
```

**Usage:**
```jsx
<Button variant="primary" size="lg">Large Primary</Button>
<Button variant="danger" loading>Deleting...</Button>
<Button variant="outline" icon={<Icon />} iconPosition="right">
  With Icon
</Button>
```

## 11. Button Best Practices

### Do:
- Use consistent padding and sizing across buttons
- Maintain sufficient contrast between text and background
- Include focus states for keyboard navigation
- Use semantic HTML (`<button>` for actions, `<a>` for navigation)
- Provide loading states for async actions

### Don't:
- Use buttons for navigation (use `<a>` tags instead)
- Make buttons too small on mobile (min 44px touch target)
- Forget disabled states
- Overcomplicate with too many variants

## 12. Button Utility Classes

| Property | Classes | Example |
|----------|---------|---------|
| Background | `bg-{color}-{shade}` | `bg-blue-500` |
| Text | `text-{color}-{shade}` | `text-white` |
| Padding | `p-*, px-*, py-*` | `py-2 px-4` |
| Border | `border, border-{color}` | `border border-gray-300` |
| Radius | `rounded, rounded-*` | `rounded-lg` |
| Hover | `hover:bg-*, hover:text-*` | `hover:bg-blue-600` |
| Focus | `focus:outline-none, focus:ring-*` | `focus:ring-2 focus:ring-blue-300` |
| Disabled | `opacity-50, cursor-not-allowed` | `opacity-50 cursor-not-allowed` |
| Transition | `transition-*` | `transition-colors` |

## Summary

Tailwind makes button styling simple and consistent. Remember to:
1. **Use semantic HTML** - `<button>` for actions, `<a>` for navigation
2. **Include all states** - hover, focus, active, disabled
3. **Ensure accessibility** - proper contrast, focus rings, ARIA labels
4. **Maintain consistency** - use the same sizing and spacing patterns
5. **Consider mobile** - adequate touch targets, responsive sizing

Buttons are your primary interaction elements - make them clear, accessible, and visually appealing with Tailwind's utility classes.
