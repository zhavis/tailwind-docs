# Responsive Design

Tailwind CSS follows a **mobile-first** approach to responsive design. This means that styles are applied for mobile screens by default, and you can add modifiers for larger screens. The framework provides responsive breakpoints that you can use to adjust your design at different screen sizes.

---

## 1. Breakpoints

Tailwind comes with five default breakpoints:

| Prefix | Minimum Width | Description |
|--------|---------------|-------------|
| `sm:`  | 640px         | Small screens |
| `md:`  | 768px         | Medium screens |
| `lg:`  | 1024px        | Large screens |
| `xl:`  | 1280px        | Extra large screens |
| `2xl:` | 1536px        | 2X large screens |

---

## 2. Mobile-First Approach

By default, styles apply to **all screen sizes**. You add responsive prefixes to apply styles **from that breakpoint and up**.

```html
<!-- Mobile: column, Tablet+: row -->
<div class="flex flex-col md:flex-row gap-4">
  <div class="w-full md:w-1/2 bg-blue-200 p-4">Column 1</div>
  <div class="w-full md:w-1/2 bg-blue-300 p-4">Column 2</div>
</div>
```

**Mobile-first rule:** 
- Start with mobile styles (no prefix)
- Add `sm:`, `md:`, `lg:` etc. for larger screens
- Styles cascade upward

---

## 3. Responsive Utilities Examples

### Text Size
```html
<h1 class="text-lg sm:text-xl md:text-2xl lg:text-3xl xl:text-4xl">
  Responsive Heading
</h1>
```

### Layout Changes
```html
<!-- Mobile: stacked, Desktop: side-by-side -->
<div class="block md:flex gap-4">
  <div class="w-full md:w-1/3 p-4 bg-gray-100">
    Sidebar (hidden on mobile)
  </div>
  <div class="w-full md:w-2/3 p-4 bg-white">
    Main content
  </div>
</div>
```

### Visibility Control
```html
<div class="hidden md:block">
  Visible only on medium screens and larger
</div>

<div class="block md:hidden">
  Visible only on mobile (hidden on md+)
</div>
```

### Spacing Adjustments
```html
<div class="p-4 md:p-8 lg:p-12">
  Padding increases with screen size
</div>
```

## 4. Complex Responsive Layout Example

```html
<div class="container mx-auto px-4">
  <!-- Grid that changes columns based on screen size -->
  <div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4">
    <div class="bg-white p-4 rounded shadow">
      <h3 class="text-base lg:text-lg font-semibold">Item 1</h3>
      <p class="text-sm md:text-base">Content adjusts with screen size</p>
    </div>
    <div class="bg-white p-4 rounded shadow">
      <h3 class="text-base lg:text-lg font-semibold">Item 2</h3>
      <p class="text-sm md:text-base">Content adjusts with screen size</p>
    </div>
    <div class="bg-white p-4 rounded shadow">
      <h3 class="text-base lg:text-lg font-semibold">Item 3</h3>
      <p class="text-sm md:text-base">Content adjusts with screen size</p>
    </div>
  </div>

  <!-- Navigation that changes layout -->
  <nav class="flex flex-col md:flex-row justify-between items-center py-4">
    <div class="mb-4 md:mb-0">
      <span class="text-xl font-bold">Logo</span>
    </div>
    <div class="space-x-4">
      <a href="#" class="block md:inline-block mb-2 md:mb-0">Home</a>
      <a href="#" class="block md:inline-block mb-2 md:mb-0">About</a>
      <a href="#" class="block md:inline-block">Contact</a>
    </div>
  </nav>
</div>
```

## 5. Custom Breakpoints

You can customize breakpoints in `tailwind.config.js`:

```javascript
module.exports = {
  theme: {
    screens: {
      'sm': '640px',
      'md': '768px',
      'lg': '1024px',
      'xl': '1280px',
      '2xl': '1536px',
      // Custom breakpoints
      'tablet': '640px',
      'laptop': '1024px',
      'desktop': '1280px',
    },
  },
}
```

Then use them:
```html
<div class="hidden laptop:block">
  Visible only on laptop screens and larger
</div>
```

## 6. Responsive Design Patterns

### 1. **Container Class**
```html
<div class="container mx-auto px-4">
  Content centered with responsive padding
</div>
```

### 2. **Responsive Images**
```html
<img src="image.jpg" 
     class="w-full h-auto max-w-sm md:max-w-md lg:max-w-lg"
     alt="Responsive image">
```

### 3. **Responsive Buttons**
```html
<button class="w-full sm:w-auto px-4 py-2 bg-blue-500 text-white rounded">
  Responsive Button
</button>
```

### 4. **Responsive Typography**
```html
<p class="text-sm md:text-base lg:text-lg leading-tight md:leading-normal">
  Text that becomes larger and more readable on bigger screens
</p>
```

## 7. Testing Responsive Design

### In Development:
1. Use browser DevTools device toolbar
2. Test all breakpoints
3. Check touch interactions

### Best Practices:
- Always design mobile-first
- Test on actual devices when possible
- Use fluid typography and spacing
- Consider touch targets on mobile (min 44px Ã— 44px)

| Task | Mobile-First Approach | Example |
|------|----------------------|---------|
| Layout | Stack columns | `flex flex-col md:flex-row` |
| Text | Smaller text first | `text-base lg:text-lg` |
| Spacing | Less padding first | `p-4 md:p-6 lg:p-8` |

| Width | Full width first | `w-full md:w-1/2` |

## 9. Example: Complete Responsive Card

```html
<div class="bg-white rounded-lg shadow-sm md:shadow-lg p-4 md:p-6 lg:p-8">
  <div class="flex flex-col md:flex-row md:items-center gap-4">
    <div class="w-full md:w-1/3">
      <img src="image.jpg" class="w-full h-48 md:h-64 object-cover rounded" alt="">
    </div>
    <div class="w-full md:w-2/3">
      <h3 class="text-xl md:text-2xl font-bold mb-2">Card Title</h3>
      <p class="text-gray-600 mb-4 text-sm md:text-base">
        This content is optimized for all screen sizes. On mobile, it's a single column with smaller text. On tablet and desktop, it becomes a two-column layout with larger typography.
      </p>
      <button class="w-full sm:w-auto px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600">
        Responsive Button
      </button>
    </div>
  </div>
</div>
```

## Key Takeaways

1. **Mobile-first**: Start with mobile styles, enhance for larger screens
2. **Breakpoint prefixes**: Use `sm:`, `md:`, `lg:`, `xl:`, `2xl:` 
3. **Fluid design**: Use relative units and responsive utilities
4. **Test thoroughly**: Check all breakpoints and interactions
5. **Performance**: Use `hidden`/`block` wisely to avoid loading unnecessary content

Responsive design with Tailwind is intuitive once you understand the mobile-first approach and breakpoint system. The utility classes make it easy to create designs that work beautifully on any device.
